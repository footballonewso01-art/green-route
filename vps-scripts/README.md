# GreenRoute VPS Scripts

Эта папка содержит всё необходимое для развертывания проекта на чистом Linux-сервере (VPS) и его дальнейшего обслуживания.

## Файлы
- `setup.sh` — Скрипт первоначальной настройки. Устанавливает Docker и запускает контейнеры.
- `deploy.sh` — Скрипт для ребилда образов и применения изменений (без даунтайма).
- `update.sh` — Скрипт для автоматического скачивания обновлений из Git (`git pull`) и запуска `deploy.sh`.
- `backup.sh` — Безопасно делает бэкап базы данных (через `tar`, временно ставя контейнер на паузу).
- `restore.sh` — Восстанавливает базу из бэкапа.
- `docker-compose.yml` — Файл поднятия инфраструктуры: PocketBase бэкенд и Caddy (реверс-прокси).
- `Caddyfile` — Настройки домена и автоматических SSL/HTTPS сертификатов.

## 🚀 Инструкция по переезду
1. **На вашем текущем ПК (или Fly.io):** Сделайте бэкап базы данных (через админку PocketBase или скачав `pocketbase/pb_data`).
2. **На новом сервере:** 
   - Скопируйте всю папку проекта на сервер (например, в `/opt/greenroute/`).
   - Положите распакованный бэкап базы в папку `pocketbase/pb_data/`.
3. **Настройка домена:** Откройте `vps-scripts/Caddyfile` и замените `api.yourdomain.com` на свой реальный домен. Убедитесь, что А-запись домена указывает на IP нового сервера.
4. **Запуск:** Перейдите в папку `vps-scripts` на сервере и выполните:
   ```bash
   chmod +x *.sh
   sudo ./setup.sh
   ```

## 📦 Работа с бэкапами
- Чтобы сделать бэкап: `./backup.sh`. Файл появится в `pocketbase/pb_data/backups_vps/`.
- Чтобы восстановить: `./restore.sh ../pocketbase/pb_data/backups_vps/имя_файла.tar.gz`.

## 🔄 Обновление проекта
Когда вы залили новые изменения в GitHub, просто зайдите на сервер в папку `vps-scripts` и выполните:
```bash
./update.sh
```
Он сам стянет изменения и перезапустит нужные контейнеры.
